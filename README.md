# Webアプリケーション脆弱性診断講座 デモサイト

このリポジトリは、**「Webアプリケーション脆弱性診断講座」向けの学習用デモ**です。  
意図的に脆弱な実装を含むミニアプリを使って、代表的なWeb脆弱性の挙動と診断ポイントを体験できます。

> [!WARNING]
> 本デモには、教育目的で脆弱性が意図的に実装されています。  
> **インターネットへ公開しないこと**、**本番データを使わないこと**、**隔離された検証環境でのみ実行すること**を厳守してください。

---

## 1. このデモで学べること

- 入力値検証不足がどのように攻撃へつながるか
- 認証・セッション・CSRF対策の不備がどのように悪用されるか
- HTTPヘッダーやテンプレート処理など、見落としがちな攻撃面
- 「再現 → 原因理解 → 対策検討」の基本的な診断学習サイクル

---

## 2. 収録されている主なデモ

`demo/index.html` から各デモへアクセスできます。

- SQL Injection (`demo/SQLi`)
- Command Injection (`demo/Commandi`)
- Cross-Site Scripting / XSS (`demo/XSS`)
- HTTP Header Injection (`demo/HTTPheaderi`)
- Path Traversal (`demo/PathTraversal`)
- CSS Injection (`demo/CSSi`)
- Relative Path Overwrite / RPO (`demo/RPO`)
- Server-Side Template Injection / SSTI (`demo/SSTi`)
- CSRF (`demo/CSRF`)
- Set-Cookie / HttpOnly挙動確認 (`demo/setcookie.php`)
- WebSocket関連デモ (`wstest`, `wstest/CSWSH.html`)

---

## 3. 事前準備（推奨）

講座用途では、以下を満たすローカル検証環境を推奨します。

- PHP が実行できるWebサーバー（Apache / nginx + php-fpm など）
- MySQL (または互換DB)
- Perl（`demo/HTTPheaderi/*.cgi` を動かす場合）

### DB初期データ

本リポジトリには `mysql.dump` が含まれており、データベース名は `vuln` 想定です。  
各デモのDB接続設定（例: `root` / `toor`）に合わせて、ローカル環境へインポートして利用してください。

---

## 4. 使い方（講座向けの進め方）

1. 受講者に対象脆弱性の概要を説明
2. デモ画面の通常操作を確認
3. PoC入力例や改変ペイロードで挙動を再現
4. サーバー側コードを読んで原因を特定
5. 修正方針（入力検証、エスケープ、トークン検証、権限チェック等）を議論

この順序で進めると、単なる「攻撃再現」で終わらず、診断・防御観点まで整理しやすくなります。
